# RavenStore Installation Guide

## Prerequisites

- Python 3.12+
- Node.js 22+
- PostgreSQL client tools (`pg_dump` and `pg_restore`)
- Redis 7+
- Supabase CLI for Storage bucket migrations
- Docker for container verification

## Secret rotation before setup

The Telegram token, Supabase service-role key, and database password previously shared outside the
provider dashboards must be rotated before use. Rotate the bot token with BotFather, rotate the
service-role key in Supabase API settings, and reset the database password in Supabase Database
settings. Do not paste the replacements into source files, GitHub issues, or deployment logs.

RavenStore uses its own JWT authentication. `JWT_SECRET_KEY` and `JWT_REFRESH_SECRET_KEY` are random
RavenStore secrets generated by `scripts/bootstrap_env.py`; they are not the Supabase Auth JWT secret.
The Supabase JWT secret is therefore not required by this architecture.

## Obtain the Supabase connection strings

Open the Supabase project and click **Connect**. For Render runtime traffic, copy the **Session pooler**
connection string because Render needs IPv4 connectivity. Change its scheme from `postgresql://` to
`postgresql+asyncpg://` and URL-encode reserved characters in the password. Store that value as
`DATABASE_URL`.

For `DATABASE_DIRECT_URL`, use the direct connection only when the deployment network supports IPv6
or the project has the IPv4 add-on. Otherwise use the Session pooler for migrations and backups too.
Never use the transaction pooler for the persistent SQLAlchemy worker connection.

## Local setup

Run the secure interactive environment bootstrap after rotating credentials:

```powershell
python scripts/bootstrap_env.py
```

It writes ignored files at:

- `apps/api/.env`
- `apps/bot/.env`
- `apps/web/.env.local`
- `apps/storefront/.env.local`

Internal JWT, encryption, metrics, backup, webhook, and service credentials are generated with a
cryptographic random source. External credentials are read with hidden prompts. The script starts
payments, uploads, and backups disabled until their verification/scanning dependencies are configured.

Install and migrate:

```powershell
cd apps/api
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -e ".[dev]"
alembic upgrade head
python -m app.seed

cd ../bot
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -e ".[dev]"

cd ../web
npm ci

cd ../storefront
npm ci
```

Apply the private/public Storage bucket definitions with the Supabase CLI:

```powershell
supabase link --project-ref YOUR_PROJECT_REF
supabase db push --include-all
```

## Local processes

Run Redis, then start these processes in separate terminals:

```powershell
cd apps/api; .venv\Scripts\python -m uvicorn app.main:app --reload --port 8000
cd apps/api; .venv\Scripts\python -m app.workers.runner
cd apps/bot; .venv\Scripts\python -m raven_bot.main
cd apps/web; npm run dev -- --port 3000
cd apps/storefront; npm run dev -- --port 3001
```

Use polling mode only for local bot development. Production must use Telegram webhook mode.
